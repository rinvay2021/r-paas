# 搜索表单查询条件配置说明

## 一、功能概述

搜索表单查询条件配置功能允许为每个字段设置 SQL 查询条件，并支持自定义默认值或系统输入，实现灵活的数据查询配置。

### 核心特性

- **字段类型过滤** - 自动过滤不可查询的字段类型（文件上传、图片上传、富文本）
- **SQL 查询条件** - 根据字段类型提供对应的 SQL 查询条件（等于、包含、大于等）
- **默认值选择** - 支持"自定义"和"系统输入"两种方式
- **动态输入组件** - 根据字段类型和查询条件动态渲染默认值输入组件
- **范围查询支持** - 支持数字、日期、时间的范围查询

## 二、配置项说明

### 2.1 表格列说明

| 列名 | 说明 | 是否必填 |
|-----|------|---------|
| **排序** | 拖拽排序，控制字段在搜索表单中的显示顺序 | - |
| **字段名称** | 字段编码，自动填充，不可编辑 | 必填 |
| **显示名称** | 在搜索表单中显示的名称 | 必填 |
| **可选查询条件** | SQL 查询条件，根据字段类型自动提供可选项 | 必填 |
| **默认值选择** | 选择"自定义"或"系统输入" | 必填 |
| **默认值** | 当选择"自定义"时可输入，"系统输入"时禁用 | 可选 |
| **提示语** | 搜索表单中该字段的提示文本 | 可选 |
| **是否显示** | 是否在搜索表单中显示该字段 | 必填 |
| **操作** | 删除该字段 | - |

### 2.2 默认值选择说明

#### 自定义（custom）
- 用户可以在配置时预设默认值
- 默认值输入框根据字段类型渲染对应的组件
- 用户打开搜索表单时，该字段会自动填充预设的默认值

**使用场景：**
- 常用的查询条件（如：状态="进行中"）
- 固定的日期范围（如：创建时间=最近7天）
- 常见的数值范围（如：年龄=18-60）

#### 系统输入（system）
- 默认值输入框禁用，不允许预设默认值
- 用户打开搜索表单时，该字段为空，需要用户手动输入
- 适合需要用户每次输入不同值的场景

**使用场景：**
- 需要用户每次输入的条件（如：订单号）
- 动态变化的条件（如：今天的日期）
- 不确定默认值的字段

## 三、不支持搜索的字段类型

以下字段类型**不能**用于搜索表单：

| 字段类型 | 枚举值 | 不支持原因 |
|---------|--------|-----------|
| **文件上传** | `FieldTypeEnum.FileUpload` | 文件对象无法作为查询条件 |
| **图片上传** | `FieldTypeEnum.ImageUpload` | 图片对象无法作为查询条件 |
| **富文本** | `FieldTypeEnum.Text_Rich` | HTML 内容过于复杂，不适合查询 |

**实现逻辑：**
```typescript
import { FieldTypeEnum } from '@/pages/meta/components/BaseField/type';

export const UNSEARCHABLE_FIELD_TYPES = [
  FieldTypeEnum.FileUpload,
  FieldTypeEnum.ImageUpload,
  FieldTypeEnum.Text_Rich,
];

export function isFieldSearchable(fieldType: FieldTypeEnum): boolean {
  return !UNSEARCHABLE_FIELD_TYPES.includes(fieldType);
}
```

## 四、字段类型与 SQL 查询条件映射

### 4.1 文本类型字段

**字段类型：**
- `FieldTypeEnum.Text` - 文本
- `FieldTypeEnum.Textarea` - 多行文本

**可用 SQL 查询条件：**

| 条件 | SQL 操作符 | 说明 | SQL 示例 |
|------|-----------|------|---------|
| 等于 | `=` | 精确匹配 | `WHERE name = '张三'` |
| 不等于 | `!=` | 排除特定值 | `WHERE status != '已删除'` |
| 包含 | `like` | 模糊匹配 | `WHERE description LIKE '%重要%'` |
| 为空 | `is_null` | 字段值为空 | `WHERE remark IS NULL` |
| 不为空 | `is_not_null` | 字段值不为空 | `WHERE remark IS NOT NULL` |

**默认值输入组件：**
- 自定义：普通文本输入框
- 系统输入：禁用的文本输入框

---

### 4.2 数字类型字段

**字段类型：**
- `FieldTypeEnum.Text_Number` - 数字输入

**可用 SQL 查询条件：**

| 条件 | SQL 操作符 | 说明 | SQL 示例 |
|------|-----------|------|---------|
| 等于 | `=` | 精确匹配 | `WHERE age = 30` |
| 不等于 | `!=` | 排除特定值 | `WHERE quantity != 0` |
| 大于 | `>` | 大于某值 | `WHERE price > 100` |
| 大于等于 | `>=` | 大于等于某值 | `WHERE age >= 18` |
| 小于 | `<` | 小于某值 | `WHERE stock < 10` |
| 小于等于 | `<=` | 小于等于某值 | `WHERE amount <= 1000` |
| 范围 | `between` | 在范围内 | `WHERE age BETWEEN 20 AND 30` |
| 为空 | `is_null` | 字段值为空 | `WHERE price IS NULL` |
| 不为空 | `is_not_null` | 字段值不为空 | `WHERE price IS NOT NULL` |

**默认值输入组件：**
- 自定义 + 普通条件：数字输入框
- 自定义 + 范围条件：两个数字输入框（最小值、最大值）

## 五、数据结构

### 5.1 SQL 查询条件操作符枚举

```typescript
export enum SqlConditionOperator {
  /** 等于 */
  EQUAL = '=',
  /** 不等于 */
  NOT_EQUAL = '!=',
  /** 包含（模糊查询） */
  LIKE = 'like',
  /** 大于 */
  GREATER_THAN = '>',
  /** 大于等于 */
  GREATER_THAN_OR_EQUAL = '>=',
  /** 小于 */
  LESS_THAN = '<',
  /** 小于等于 */
  LESS_THAN_OR_EQUAL = '<=',
  /** 范围 */
  BETWEEN = 'between',
  /** 在列表中 */
  IN = 'in',
  /** 不在列表中 */
  NOT_IN = 'not_in',
  /** 为空 */
  IS_NULL = 'is_null',
  /** 不为空 */
  IS_NOT_NULL = 'is_not_null',
}
```

### 5.2 搜索表单字段 DTO

```typescript
export interface SearchFormFieldDto {
  /** 字段名称（字段编码） */
  fieldName: string;

  condition: string;
  /** 默认值选择方式：custom-自定义，system-系统输入 */
  defaultValueType?: 'custom' | 'system';
  /** 默认值 */
  defaultValue?: string;

  placeholder?: string;

  isVisible?: boolean;

```

## 八、注意事项

### 8.1 默认值选择方式

- **自定义**：适合有固定默认值的场景
- **系统输入**：适合需要用户每次输入的场景
- 切换为"系统输入"时，默认值会自动清空

### 8.2 查询条件切换

- 切换查询条件时，默认值会自动清空
- 避免因条件变化导致默认值格式不匹配

### 8.3 默认值格式

- 普通条件：单个值
- 范围条件（between）：数组 `[min, max]`
- 多选条件（in/not_in）：数组 `[value1, value2, ...]`

### 8.4 字段类型映射

- 添加字段时，系统会自动建立字段名到字段类型的映射
- 确保字段类型正确，否则查询条件和默认值组件可能无法正常工作

## 九、常见问题

### Q1：为什么某些字段不能添加到搜索表单？
**A：** 可能的原因：
1. 字段类型是 FileUpload、ImageUpload 或 Text_Rich
2. 字段已被添加到搜索表单中
3. 字段已被删除或禁用

### Q2："默认值选择"有什么作用？
**A：**
- **自定义**：可以预设默认值，用户打开搜索表单时自动填充
- **系统输入**：不预设默认值，用户打开搜索表单时需要手动输入

### Q3：什么时候选择"自定义"，什么时候选择"系统输入"？
**A：**
- **自定义**：常用的查询条件，如状态="进行中"、日期=最近7天
- **系统输入**：每次查询值不同的字段，如订单号、用户名

### Q4：切换查询条件后，默认值为什么清空了？
**A：** 不同查询条件的默认值格式可能不同，自动清空避免格式错误。例如：
- 等于条件：单个值
- 范围条件：两个值（最小值、最大值）

### Q5：为什么"系统输入"时默认值输入框是灰色的？
**A：** 这是正常的，"系统输入"表示不预设默认值，由用户在搜索表单中手动输入。
