# 表单联动功能技术文档

## 一、功能概述

表单联动功能允许根据某个字段的值，动态控制其他字段的显示、隐藏、必填、只读等状态，实现字段之间的智能联动。

### 核心特性

- **可视化配置** - 无需编写代码，通过界面配置联动规则
- **实时响应** - 表单值变化时，自动执行联动规则
- **自动恢复** - 条件不满足时，自动执行反向操作
- **冲突处理** - 多个规则作用于同一字段时，按优先级处理
- **字段过滤** - 只显示表单中存在的、支持联动的字段

## 二、联动规则配置

### 2.1 规则结构

一条联动规则包含：
- **触发条件** - 当某个字段满足某个条件时触发
- **执行动作** - 对其他字段执行的操作（可配置多个动作）

```typescript
{
  id: "rule-123",
  condition: {
    field: "hasSpouse",      // 触发字段
    operator: "==",          // 操作符
    value: "是"              // 条件值
  },
  actions: {
    show: ["spouseName"],    // 显示字段
    require: ["spouseName"]  // 必填字段
  }
}
```

### 2.2 支持的联动类型

| 联动类型 | 说明 | 使用场景 |
|---------|------|---------|
| **显示字段** (show) | 条件满足时显示指定字段 | 选择"是否有配偶"为"是"时，显示配偶信息 |
| **隐藏字段** (hide) | 条件满足时隐藏指定字段 | 选择"配送方式"为"自提"时，隐藏收货地址 |
| **必填字段** (require) | 条件满足时设为必填 | 选择"是否开发票"为"是"时，发票信息必填 |
| **非必填** (unrequire) | 条件满足时设为非必填 | 选择"是否开发票"为"否"时，发票信息非必填 |
| **只读字段** (readonly) | 条件满足时设为只读 | 订单状态为"已完成"时，所有字段只读 |
| **可编辑** (editable) | 条件满足时设为可编辑 | 订单状态为"待处理"时，字段可编辑 |
| **清空字段** (clear) | 条件满足时清空字段值 | 切换"配送方式"时，清空之前的地址信息 |

### 2.3 支持的条件操作符

| 操作符 | 说明 | 适用场景 |
|-------|------|---------|
| == | 等于 | 选择框、单选框、布尔值 |
| != | 不等于 | 排除某些特定值 |
| > | 大于 | 数字字段的范围判断 |
| < | 小于 | 数字字段的范围判断 |
| >= | 大于等于 | 金额、数量等数值判断 |
| <= | 小于等于 | 金额、数量等数值判断 |
| contains | 包含 | 文本字段的模糊匹配 |
| isEmpty | 为空 | 判断字段是否未填写 |
| isNotEmpty | 不为空 | 判断字段是否已填写 |

## 三、字段过滤规则

### 3.1 只能选择表单中存在的字段

系统会自动从表单的所有容器中提取字段列表，只有在表单中实际存在的字段才能参与联动配置。

**实现逻辑：**
```typescript
// 从所有容器中提取字段编码
const formFieldCodes = new Set(
  containers.flatMap(c => c.fields?.map(f => f.fieldCode) || [])
);

// 过滤出在表单中的字段
const availableFields = allFields.filter(field =>
  formFieldCodes.has(field.fieldCode)
);
```

### 3.2 不支持联动的字段类型

以下字段类型**不能**作为联动的触发条件或目标字段：

| 字段类型 | 枚举值 | 不支持原因 |
|---------|--------|-----------|
| **文件上传** | `FieldTypeEnum.FileUpload` | 文件上传字段，值为文件对象，不适合作为联动条件 |
| **图片上传** | `FieldTypeEnum.ImageUpload` | 图片上传字段，值为图片对象，不适合作为联动条件 |
| **富文本** | `FieldTypeEnum.Text_Rich` | 富文本编辑器，值为 HTML 内容，过于复杂 |

**实现逻辑：**
```typescript
import { FieldTypeEnum } from '@/pages/meta/components/BaseField/type';

const UNSUPPORTED_FIELD_TYPES = [
  FieldTypeEnum.FileUpload,
  FieldTypeEnum.ImageUpload,
  FieldTypeEnum.Text_Rich,
];

const availableFields = allFields.filter(field => {
  if (!formFieldCodes.has(field.fieldCode)) return false;
  if (UNSUPPORTED_FIELD_TYPES.includes(field.fieldType)) {
    return false;
  }
  return true;
});
```

### 3.3 数据源字段特殊处理

对于有固定选项的字段（数据源字段），条件值可以从下拉框选择，而不是手动输入。

**数据源字段类型：**

| 字段类型 | 枚举值 | 说明 |
|---------|--------|------|
| **下拉单选** | `FieldTypeEnum.SingleSelect` | 下拉选择框（单选） |
| **下拉多选** | `FieldTypeEnum.MultipleSelect` | 下拉选择框（多选） |
| **单选按钮** | `FieldTypeEnum.SingleRadio` | 单选按钮组 |
| **复选框** | `FieldTypeEnum.MultipleCheckbox` | 复选框组 |

**实现逻辑：**
```typescript
import { FieldTypeEnum } from '@/pages/meta/components/BaseField/type';

const DATA_SOURCE_FIELD_TYPES = [
  FieldTypeEnum.SingleSelect,
  FieldTypeEnum.MultipleSelect,
  FieldTypeEnum.SingleRadio,
  FieldTypeEnum.MultipleCheckbox,
];

const isDataSourceField = (fieldCode: string) => {
  const field = fields.find(f => f.fieldCode === fieldCode);
  return field && DATA_SOURCE_FIELD_TYPES.includes(field.fieldType as FieldTypeEnum);
};

// 渲染条件值输入
{isDataSourceField(rule.condition.field) ? (
  <Select placeholder="选择条件值" options={getFieldValueOptions(...)} />
) : (
  <Input placeholder="输入条件值" />
)}
```

## 四、高级特性

### 4.1 反向联动（自动恢复）

当条件不满足时，系统会自动执行反向操作：

| 正向操作 | 反向操作 |
|---------|---------|
| 显示 | 隐藏 |
| 隐藏 | 显示 |
| 必填 | 非必填 |
| 非必填 | 必填 |
| 只读 | 可编辑 |
| 可编辑 | 只读 |

**示例：**
```
规则：当 hasSpouse == "是" 时，显示 spouseName
效果：
  - hasSpouse = "是" → spouseName 显示
  - hasSpouse = "否" → spouseName 隐藏（自动恢复）
```

### 4.2 冲突处理（优先级规则）

当多个规则作用于同一个字段时，按以下优先级处理：

- **显隐规则**：隐藏优先（任何一个规则要求隐藏就隐藏）
- **必填规则**：必填优先（任何一个规则要求必填就必填）
- **只读规则**：只读优先（任何一个规则要求只读就只读）
- **清空规则**：按规则定义顺序执行

## 五、数据结构

### 5.1 存储格式

联动规则直接存储在表单配置的 `linkageSettings` 字段中，格式为数组：

```typescript
{
  formConfig: {
    helpSettings: {...},
    layoutSettings: {...},
    linkageSettings: [  // 直接是数组
      {
        id: "rule-123",
        condition: {
          field: "hasSpouse",
          operator: "==",
          value: "是"
        },
        actions: {
          show: ["spouseName", "spouseAge"],
          require: ["spouseName"]
        }
      }
    ]
  }
}
```

### 5.2 类型定义

```typescript
// 联动规则
interface LinkageRule {
  id: string;                    // 规则ID（自动生成）
  condition: LinkageCondition;   // 触发条件
  actions: LinkageActions;       // 执行动作
}

// 触发条件
interface LinkageCondition {
  field: string;                 // 字段编码
  operator: ConditionOperator;   // 操作符
  value: any;                    // 条件值
}

// 执行动作
interface LinkageActions {
  show?: string[];        // 显示的字段列表
  hide?: string[];        // 隐藏的字段列表
  require?: string[];     // 必填的字段列表
  unrequire?: string[];   // 非必填的字段列表
  readonly?: string[];    // 只读的字段列表
  editable?: string[];    // 可编辑的字段列表
  clear?: string[];       // 清空的字段列表
}

// 联动配置（直接是数组）
type LinkageSettings = LinkageRule[];
```

### 6.2 关键技术点

#### 6.2.1 React 状态管理

使用函数式更新避免闭包陷阱：

```typescript
// ❌ 错误：可能基于过时的状态
setRules(rules.map(rule => ...));

// ✅ 正确：基于最新的状态
setRules(prevRules => prevRules.map(rule => ...));
```

合并状态更新避免覆盖：

```typescript
// ❌ 错误：连续调用可能导致覆盖
handleUpdateCondition(rule.id, 'field', value);
handleUpdateCondition(rule.id, 'value', '');

// ✅ 正确：一次性更新多个字段
setRules(prevRules =>
  prevRules.map(r =>
    r.id === rule.id
      ? { ...r, condition: { ...r.condition, field: value, value: '' } }
      : r
  )
);
```

#### 6.2.2 运行时引擎

**核心函数：**
1. `checkCondition` - 检查条件是否满足
2. `calculateLinkage` - 计算联动结果
3. `applyLinkageResult` - 应用联动结果到表单状态
4. `executeLinkage` - 执行联动规则（主入口）

**执行流程：**
```
表单值变化
    ↓
遍历所有规则
    ↓
检查条件是否满足
    ↓
条件满足 → 执行正向动作
条件不满足 → 执行反向动作（自动恢复）
    ↓
处理冲突（优先级规则）
    ↓
更新字段状态
    ↓
清空需要清空的字段
```

#### 6.2.3 在表单预览中使用

```tsx
import { Form } from 'antd';
import { useLinkage } from './hooks/useLinkage';

function FormPreview({ formConfig, fields }) {
  const [form] = Form.useForm();

  // 使用联动 Hook
  const { isFieldVisible, isFieldRequired, isFieldReadonly } = useLinkage({
    rules: formConfig.linkageSettings || [],
    form,
    fields: fields.map(f => f.fieldCode),
  });

  return (
    <Form form={form}>
      {fields.map(field => {
        // 根据联动状态决定是否渲染字段
        if (!isFieldVisible(field.fieldCode)) {
          return null;
        }

        return (
          <Form.Item
            key={field.fieldCode}
            name={field.fieldCode}
            label={field.fieldName}
            required={isFieldRequired(field.fieldCode)}
          >
            <Input
              disabled={isFieldReadonly(field.fieldCode)}
              placeholder={`请输入${field.fieldName}`}
            />
          </Form.Item>
        );
      })}
    </Form>
  );
}
```

## 七、使用场景示例

## 八、注意事项

### 8.1 使用限制

1. **避免循环依赖**
   - 不要创建循环依赖的联动规则
   - 例如：A 控制 B，B 又控制 A

2. **性能考虑**
   - 联动规则会在表单值变化时触发
   - 过多的规则可能影响性能
   - 建议单个表单的联动规则不超过 20 条

3. **字段清空谨慎使用**
   - 清空操作会立即清除字段值
   - 用户已填写的数据会丢失
   - 建议配合确认提示使用

4. **必填联动覆盖默认设置**
   - 联动设置的必填状态会覆盖字段的默认必填设置
   - 需要注意优先级关系

### 8.2 最佳实践

1. **规则设计清晰**
   - 确保每条规则的用途明确
   - 避免过于复杂的联动逻辑

2. **条件值精确**
   - 使用精确的条件值，避免模糊匹配导致误触发

3. **动作合理**
   - 一条规则可以配置多个动作
   - 但要确保动作之间没有冲突

4. **测试充分**
   - 配置完成后，在表单预览中充分测试
   - 覆盖各种条件分支

## 九、未来扩展方向

### 9.1 第一优先级（常用功能）

1. **值联动** - 根据其他字段的值自动计算当前字段的值
   - 简单公式：`{price} * {quantity}`
   - 复制值：直接复制其他字段的值

2. **选项联动** - 根据其他字段的值动态改变下拉选项
   - 省市区三级联动
   - 分类品牌联动

### 9.2 第二优先级（高级功能）

3. **多条件组合** - 支持 AND/OR 组合多个条件
   - 当字段A=值1 **且** 字段B=值2 时
   - 当字段A=值1 **或** 字段B=值2 时

4. **校验联动** - 根据其他字段的值动态改变校验规则
   - 根据证件类型改变证件号码的校验规则

### 9.3 第三优先级（扩展功能）

5. **自定义函数** - 支持自定义 JavaScript 函数
   - 复杂的条件判断
   - 复杂的值计算

6. **API 联动** - 根据字段值调用 API
   - 根据商品ID获取价格
   - 根据地址获取邮费

## 十、常见问题

### Q1：为什么某些字段不能选择？
**A：** 可能的原因：
1. 字段不在当前表单容器中
2. 字段类型是 FILE、IMAGE 或 RICH_TEXT
3. 字段已被删除或禁用

### Q2：联动规则不生效？
**A：** 检查以下几点：
1. 条件值是否正确（注意大小写、空格）
2. 字段编码是否正确
3. 是否保存了配置
4. 浏览器控制台是否有错误

### Q3：多个规则冲突怎么办？
**A：** 系统会按优先级处理：
- 隐藏优先、必填优先、只读优先
- 如果需要特定优先级，调整规则顺序

### Q4：如何调试联动规则？
**A：** 打开浏览器控制台，查看联动引擎的日志输出。

- ✅ 无需编写代码

- ✅ 实时响应
