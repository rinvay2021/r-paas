# 表单联动功能说明文档

## 一、功能概述

表单联动功能允许根据某个字段的值，动态控制其他字段的显示、隐藏、必填、只读等状态，实现字段之间的智能联动，提升表单的交互体验和数据准确性。

## 二、功能特性

### 2.1 支持的联动类型

| 联动类型 | 说明 | 使用场景 |
|---------|------|---------|
| **显示字段** | 当条件满足时，显示指定的字段 | 选择"是否有配偶"为"是"时，显示配偶信息字段 |
| **隐藏字段** | 当条件满足时，隐藏指定的字段 | 选择"配送方式"为"自提"时，隐藏收货地址字段 |
| **必填字段** | 当条件满足时，将指定字段设为必填 | 选择"是否开发票"为"是"时，发票信息变为必填 |
| **非必填** | 当条件满足时，将指定字段设为非必填 | 选择"是否开发票"为"否"时，发票信息变为非必填 |
| **只读字段** | 当条件满足时，将指定字段设为只读 | 订单状态为"已完成"时，所有字段变为只读 |
| **可编辑** | 当条件满足时，将指定字段设为可编辑 | 订单状态为"待处理"时，字段可编辑 |
| **清空字段** | 当条件满足时，清空指定字段的值 | 切换"配送方式"时，清空之前填写的地址信息 |

### 2.2 支持的条件操作符

| 操作符 | 说明 | 适用场景 |
|-------|------|---------|
| **等于 (==)** | 字段值等于指定值 | 选择框、单选框、布尔值 |
| **不等于 (!=)** | 字段值不等于指定值 | 排除某些特定值的情况 |
| **大于 (>)** | 字段值大于指定值 | 数字字段的范围判断 |
| **小于 (<)** | 字段值小于指定值 | 数字字段的范围判断 |
| **大于等于 (>=)** | 字段值大于等于指定值 | 金额、数量等数值判断 |
| **小于等于 (<=)** | 字段值小于等于指定值 | 金额、数量等数值判断 |
| **包含 (contains)** | 字段值包含指定文本 | 文本字段的模糊匹配 |
| **为空 (isEmpty)** | 字段值为空 | 判断字段是否未填写 |
| **不为空 (isNotEmpty)** | 字段值不为空 | 判断字段是否已填写 |

### 2.3 高级特性

#### 反向联动（自动恢复）
当条件不满足时，系统会自动执行反向操作：
- 显示 → 隐藏
- 隐藏 → 显示
- 必填 → 非必填
- 非必填 → 必填
- 只读 → 可编辑
- 可编辑 → 只读

**示例：**
```
规则：当 hasSpouse == "是" 时，显示 spouseName
效果：
  - hasSpouse = "是" → spouseName 显示
  - hasSpouse = "否" → spouseName 隐藏（自动恢复）
```

#### 冲突处理（优先级规则）
当多个规则作用于同一个字段时，按以下优先级处理：
- **显隐规则**：隐藏优先（任何一个规则要求隐藏就隐藏）
- **必填规则**：必填优先（任何一个规则要求必填就必填）
- **只读规则**：只读优先（任何一个规则要求只读就只读）
- **清空规则**：按规则定义顺序执行

#### 实时响应
- 表单值变化时，实时计算并应用联动规则
- 无需手动触发，自动响应字段变化

## 三、字段过滤规则

### 3.1 可以设置联动的字段

只有**在当前表单容器中存在**的字段才能参与联动配置。系统会自动从表单的所有容器中提取字段列表。

**支持联动的字段类型：**
- ✅ **STRING** - 文本字段
- ✅ **NUMBER** - 数字字段
- ✅ **DATE** - 日期字段
- ✅ **DATETIME** - 日期时间字段
- ✅ **TIME** - 时间字段
- ✅ **TEXTAREA** - 多行文本
- ✅ **SELECT** - 下拉选择
- ✅ **MULTI_SELECT** - 多选
- ✅ **RADIO** - 单选
- ✅ **CHECKBOX** - 复选框
- ✅ **BOOLEAN** - 布尔值
- ✅ **CASCADER** - 级联选择
- ✅ **TREE_SELECT** - 树形选择
- ✅ **REFERENCE** - 关联引用

### 3.2 不支持联动的字段类型

以下字段类型**不能**作为联动的触发条件或目标字段：

| 字段类型 | 枚举值 | 不支持原因 |
|---------|--------|-----------|
| **文件上传** | `FieldTypeEnum.FileUpload` | 文件上传字段，值为文件对象，不适合作为联动条件 |
| **图片上传** | `FieldTypeEnum.ImageUpload` | 图片上传字段，值为图片对象，不适合作为联动条件 |
| **富文本** | `FieldTypeEnum.Text_Rich` | 富文本编辑器，值为 HTML 内容，过于复杂 |

**代码实现：**
```typescript
// LinkageSettingsButton/index.tsx
import { FieldTypeEnum } from '@/pages/meta/components/BaseField/type';

const UNSUPPORTED_FIELD_TYPES = [
  FieldTypeEnum.FileUpload,
  FieldTypeEnum.ImageUpload,
  FieldTypeEnum.Text_Rich,
];

const availableFields = allFields.filter(field => {
  // 必须在表单中
  if (!formFieldCodes.has(field.fieldCode)) return false;
  // 过滤不支持联动的字段类型
  if (UNSUPPORTED_FIELD_TYPES.includes(field.fieldType)) return false;
  return true;
});
```

### 3.3 数据源字段特殊处理

对于有固定选项的字段（数据源字段），条件值可以从下拉框选择：

**数据源字段类型：**

| 字段类型 | 枚举值 | 说明 |
|---------|--------|------|
| **下拉单选** | `FieldTypeEnum.SingleSelect` | 下拉选择框（单选） |
| **下拉多选** | `FieldTypeEnum.MultipleSelect` | 下拉选择框（多选） |
| **单选按钮** | `FieldTypeEnum.SingleRadio` | 单选按钮组 |
| **复选框** | `FieldTypeEnum.MultipleCheckbox` | 复选框组 |

**普通字段 vs 数据源字段：**

| 字段类型 | 条件值输入方式 | 示例 |
|---------|---------------|------|
| 普通字段（Text, Text_Number等） | 手动输入文本框 | 输入 "张三"、"100" |
| 数据源字段（SingleSelect, SingleRadio等） | 下拉框选择 | 从选项中选择 |

**代码实现：**
```typescript
// LinkageModal.tsx
import { FieldTypeEnum } from '@/pages/meta/components/BaseField/type';

const DATA_SOURCE_FIELD_TYPES = [
  FieldTypeEnum.SingleSelect,
  FieldTypeEnum.MultipleSelect,
  FieldTypeEnum.SingleRadio,
  FieldTypeEnum.MultipleCheckbox,
];

const isDataSourceField = (fieldCode: string) => {
  const field = fields.find(f => f.fieldCode === fieldCode);
  return field && DATA_SOURCE_FIELD_TYPES.includes(field.fieldType as FieldTypeEnum);
};

// 渲染条件值输入框
{isDataSourceField(rule.condition.field) ? (
  <Select placeholder="选择条件值" options={getFieldValueOptions(rule.condition.field)} />
) : (
  <Input placeholder="输入条件值" />
)}
```

## 四、使用场景示例

### 场景1：根据"是否有配偶"显示配偶信息

**业务需求：**
- 当用户选择"是否有配偶"为"是"时，显示"配偶姓名"和"配偶年龄"字段，且"配偶姓名"为必填
- 当用户选择"否"时，隐藏相关字段

**配置方法：**
1. 触发条件：当 `hasSpouse` 等于 `是`
2. 执行动作：
   - 显示字段：`spouseName`, `spouseAge`
   - 必填字段：`spouseName`

**效果：**
```
hasSpouse = "是"
  → spouseName 显示 + 必填
  → spouseAge 显示

hasSpouse = "否"
  → spouseName 隐藏 + 非必填（自动恢复）
  → spouseAge 隐藏（自动恢复）
```

### 场景2：根据配送方式显示地址信息

**业务需求：**
- 选择"配送方式"为"快递"时，显示"收货地址"和"联系电话"，且都为必填
- 选择"自提"时，隐藏地址信息

**配置方法：**
1. 触发条件：当 `deliveryType` 等于 `快递`
2. 执行动作：
   - 显示字段：`address`, `phone`
   - 必填字段：`address`, `phone`

### 场景3：根据订单金额显示优惠券

**业务需求：**
- 当订单金额 >= 100 时，显示"优惠券"字段
- 当订单金额 < 100 时，隐藏优惠券字段

**配置方法：**
1. 触发条件：当 `totalAmount` 大于等于 `100`
2. 执行动作：
   - 显示字段：`couponCode`

### 场景4：根据订单状态设置只读

**业务需求：**
- 订单状态为"已完成"或"已取消"时，所有字段变为只读
- 订单状态为"待处理"时，字段可编辑

**配置方法：**
1. 规则1 - 触发条件：当 `orderStatus` 等于 `已完成`
   - 执行动作：只读字段：`所有业务字段`

2. 规则2 - 触发条件：当 `orderStatus` 等于 `已取消`
   - 执行动作：只读字段：`所有业务字段`

### 场景5：切换选项时清空关联字段

**业务需求：**
- 当用户切换"配送方式"时，清空之前填写的地址信息

**配置方法：**
1. 触发条件：当 `deliveryType` 不等于 `快递`
2. 执行动作：
   - 清空字段：`address`, `phone`

## 五、数据结构

### 5.1 存储格式

联动规则直接存储在表单配置的 `linkageSettings` 字段中，格式为数组：

```typescript
{
  formConfig: {
    helpSettings: {...},
    layoutSettings: {...},
    linkageSettings: [  // 直接是数组
      {
        id: "rule-123",
        condition: {
          field: "hasSpouse",
          operator: "==",
          value: "是"
        },
        actions: {
          show: ["spouseName", "spouseAge"],
          require: ["spouseName"]
        }
      },
      {
        id: "rule-456",
        condition: {
          field: "deliveryType",
          operator: "==",
          value: "快递"
        },
        actions: {
          show: ["address", "phone"],
          require: ["address", "phone"]
        }
      }
    ]
  }
}
```

### 5.2 类型定义

```typescript
// 联动规则
interface LinkageRule {
  id: string;                    // 规则ID（自动生成）
  condition: LinkageCondition;   // 触发条件
  actions: LinkageActions;       // 执行动作
}

// 触发条件
interface LinkageCondition {
  field: string;                 // 字段编码
  operator: ConditionOperator;   // 操作符
  value: any;                    // 条件值
}

// 执行动作
interface LinkageActions {
  show?: string[];        // 显示的字段列表
  hide?: string[];        // 隐藏的字段列表
  require?: string[];     // 必填的字段列表
  unrequire?: string[];   // 非必填的字段列表
  readonly?: string[];    // 只读的字段列表
  editable?: string[];    // 可编辑的字段列表
  clear?: string[];       // 清空的字段列表
}

// 联动配置（直接是数组）
type LinkageSettings = LinkageRule[];
```

```
    └── LinkageModal.less             # 弹窗样式
```

### 6.2 运行时引擎

**核心函数：**
1. `checkCondition` - 检查条件是否满足
2. `calculateLinkage` - 计算联动结果
3. `applyLinkageResult` - 应用联动结果到表单状态
4. `executeLinkage` - 执行联动规则（主入口）

**执行流程：**
```
表单值变化
    ↓
遍历所有规则
    ↓
检查条件是否满足
    ↓
条件满足 → 执行正向动作
条件不满足 → 执行反向动作（自动恢复）
    ↓
处理冲突（优先级规则）
    ↓
更新字段状态
    ↓
清空需要清空的字段
```

### 6.3 在表单预览中使用

```tsx
import { Form } from 'antd';
import { useLinkage } from './hooks/useLinkage';

function FormPreview({ formConfig, fields }) {
  const [form] = Form.useForm();

  // 使用联动 Hook
  const { isFieldVisible, isFieldRequired, isFieldReadonly } = useLinkage({
    rules: formConfig.linkageSettings || [],
    form,
    fields: fields.map(f => f.fieldCode),
  });

  return (
    <Form form={form}>
      {fields.map(field => {
        // 根据联动状态决定是否渲染字段
        if (!isFieldVisible(field.fieldCode)) {
          return null;
        }

        return (
          <Form.Item
            key={field.fieldCode}
            name={field.fieldCode}
            label={field.fieldName}
            required={isFieldRequired(field.fieldCode)}
          >
            <Input
              disabled={isFieldReadonly(field.fieldCode)}
              placeholder={`请输入${field.fieldName}`}
            />
          </Form.Item>
        );
      })}
    </Form>
  );
}
```

## 七、注意事项

### 7.1 使用限制

1. **避免循环依赖**
   - 不要创建循环依赖的联动规则
   - 例如：A 控制 B，B 又控制 A

2. **性能考虑**
   - 联动规则会在表单值变化时触发
   - 过多的规则可能影响性能
   - 建议单个表单的联动规则不超过 20 条

3. **字段清空谨慎使用**
   - 清空操作会立即清除字段值
   - 用户已填写的数据会丢失
   - 建议配合确认提示使用

4. **必填联动覆盖默认设置**
   - 联动设置的必填状态会覆盖字段的默认必填设置
   - 需要注意优先级关系

### 7.2 最佳实践

1. **规则命名清晰**
   - 虽然界面上没有规则名称，但在设计时要清楚每条规则的用途

2. **条件值精确**
   - 使用精确的条件值，避免模糊匹配导致误触发

3. **动作合理**
   - 一条规则可以配置多个动作
   - 但要确保动作之间没有冲突

4. **测试充分**
   - 配置完成后，在表单预览中充分测试
   - 覆盖各种条件分支

## 八、未来扩展方向

### 8.1 第一优先级（常用功能）
1. **值联动** - 根据其他字段的值自动计算当前字段的值
   - 简单公式：`{price} * {quantity}`
   - 复制值：直接复制其他字段的值

2. **选项联动** - 根据其他字段的值动态改变下拉选项
   - 省市区三级联动
   - 分类品牌联动

### 8.2 第二优先级（高级功能）
3. **多条件组合** - 支持 AND/OR 组合多个条件
   - 当字段A=值1 **且** 字段B=值2 时
   - 当字段A=值1 **或** 字段B=值2 时

4. **校验联动** - 根据其他字段的值动态改变校验规则
   - 根据证件类型改变证件号码的校验规则

### 8.3 第三优先级（扩展功能）
5. **自定义函数** - 支持自定义 JavaScript 函数
   - 复杂的条件判断
   - 复杂的值计算

6. **API 联动** - 根据字段值调用 API
   - 根据商品ID获取价格
   - 根据地址获取邮费

## 九、常见问题

### Q1：为什么某些字段不能选择？
**A：** 可能的原因：
1. 字段不在当前表单容器中
2. 字段类型是 FILE、IMAGE 或 RICH_TEXT
3. 字段已被删除或禁用

### Q2：联动规则不生效？
**A：** 检查以下几点：
1. 条件值是否正确（注意大小写、空格）
2. 字段编码是否正确
3. 是否保存了配置
4. 浏览器控制台是否有错误

### Q3：多个规则冲突怎么办？
**A：** 系统会按优先级处理：
- 隐藏优先、必填优先、只读优先
- 如果需要特定优先级，调整规则顺序

### Q4：如何调试联动规则？
**A：** 打开浏览器控制台，查看联动引擎的日志输出。

- ✅ 无需编写代码

- ✅ 实时响应
- ✅ 自动恢复
- ✅ 冲突处理
- ✅ 易于维护

**适用场景：**
- 复杂表单的字段联动
